**핵심 요약**

- “토큰(token)”은 LLM에서 새로 만든 개념이 아니라 **자연어처리(NLP)에서 오래전부터 쓰이던 단위**다.
    
- 1950~60년대 **초기 컴퓨터 언어처리 연구**에서 등장했다.
    
- 목적은 **문장을 컴퓨터가 처리 가능한 단위로 분해**하는 것.
    
- 초기에는 **단어 기반 토큰**이었다.
    
- 이후 딥러닝 시대에 **subword 토큰(BPE, SentencePiece)** 방식이 등장하면서 현재 LLM 구조가 만들어졌다.
    

---

# 1. 토큰 개념의 시작 (1950~1970)

컴퓨터가 텍스트를 처리하려면 먼저 해야 하는 일이 있다.

문장

```
The cat sat on the mat
```

컴퓨터 입장에서는

```
문자열 덩어리
```

일 뿐이다.

그래서 연구자들이 만든 첫 단계가

```
Tokenization
```

이다.

즉

```
문장 → 단어 단위 분리
```

예

```
The | cat | sat | on | the | mat
```

이때 각각을

```
token
```

이라고 불렀다.

이 개념은 **컴파일러 연구에서 온 것**이다.

---

# 2. 사실 토큰은 컴파일러에서 먼저 등장

프로그래밍 언어 컴파일러는

코드를 이렇게 처리한다.

예

```
int x = 10;
```

분해

```
int | x | = | 10 | ;
```

각각이

```
token
```

이다.

이 단계는

```
lexical analysis
```

라고 불린다.

이 개념이 그대로 **자연어 처리(NLP)**로 넘어왔다.

---

# 3. 단어 토큰의 한계

초기 NLP는 이렇게 했다.

```
단어 = 토큰
```

하지만 문제가 생겼다.

### 문제 1 — 희귀 단어

```
Pneumonoultramicroscopicsilicovolcanoconiosis
```

모델이 본 적 없는 단어.

### 문제 2 — 형태 변화

```
run
runs
running
ran
```

모두 다른 단어.

### 문제 3 — 언어 다양성

한국어

```
먹었습니다
먹었다
먹는다
```

단어 수가 폭발한다.

이걸

```
vocabulary explosion
```

이라고 한다.

---

# 4. 해결책: subword 토큰

2010년대에 중요한 아이디어가 등장했다.

```
단어가 아니라 "부분 단어"를 토큰으로 쓰자
```

대표 방법

```
BPE (Byte Pair Encoding)
```

예

```
internationalization
```

→

```
inter
nation
al
ization
```

장점

```
희귀 단어도 분해 가능
```

---

# 5. Transformer 이후 (2017~)

2017년

**Transformer 논문**

이후 LLM 구조가 만들어졌다.

이 모델 구조에서는

텍스트를 반드시

```
정수 sequence
```

로 바꿔야 한다.

그래서 과정이

```
텍스트
↓
토큰화
↓
토큰 ID
↓
embedding
↓
신경망
```

이 된다.

예

```
나는 오늘 책을 읽었다
```

→

```
[14521, 921, 5510, 8332, ...]
```

---

# 6. 토큰이 중요한 이유

LLM에게 텍스트는 사실

```
토큰 시퀀스
```

일 뿐이다.

모델이 실제로 하는 일은

```
다음 토큰 예측
```

이다.

예

```
The cat sat on the
```

→ 다음 토큰

```
mat
floor
chair
```

확률 계산.

---

# 7. 그래서 모든 것이 토큰 기준

LLM에서 중요한 것들은 전부

```
토큰
```

기준이다.

예

- 컨텍스트 길이
    
- 모델 비용
    
- 속도
    
- 메모리
    

예

```
128k context
```

의 의미

```
128,000 tokens
```
