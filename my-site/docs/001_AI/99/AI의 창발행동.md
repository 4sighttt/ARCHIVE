# Generative Agents 논문 구조 분석

## 1. 논문의 핵심 주장 (Core Claim)

이 논문의 핵심 주장은 다음과 같다.

**대형 언어 모델(LLM)에 “기억(memory)·반성(reflection)·계획(planning)” 구조를 결합하면 인간처럼 지속적이고 일관된 행동을 하는 AI 에이전트를 만들 수 있다.**

기존 LLM은 **단일 시점의 반응은 자연스럽지만 장기적 행동 일관성이 없다.**  
이 논문은 이를 해결하기 위해 **에이전트 아키텍처를 설계**한다.

핵심 주장:

- LLM + 장기 기억 시스템 → 지속적 캐릭터 유지
    
- 기억 + 반성 + 계획 → 장기 행동 일관성
    
- 여러 에이전트가 상호작용하면 **사회적 행동이 창발적으로 발생**
    

실험에서는 25개의 AI 캐릭터가 가상 마을에서 생활하며  
정보 확산, 관계 형성, 이벤트 조직 등이 **자발적으로 발생**한다.

---

# 2. 연구 문제 정의 (Research Question)

논문이 해결하려는 핵심 문제는 다음이다.

**“인간처럼 지속적인 행동과 사회적 상호작용을 하는 AI 에이전트를 만들 수 있는가?”**

구체적인 문제:

1. LLM은 **단기 대화에는 강하지만 장기 행동 모델이 없다**
    
2. 과거 경험을 축적하는 **메모리 구조가 부족**
    
3. 여러 에이전트 사이에서 **사회적 상호작용을 유지하기 어렵다**
    

따라서 연구 질문은 다음과 같이 정리된다.

> LLM을 기반으로 장기 기억과 계획 능력을 가진 **인간 행동 시뮬레이션 에이전트**를 만들 수 있는가?

# 4. 방법론 분석

논문이 제안한 **Generative Agent Architecture**는 세 가지 핵심 모듈로 구성된다.

### 1) Memory Stream

모든 경험을 자연어로 저장한다.

예:

```
John Lin is cooking breakfast
Maria is studying chemistry
Tom said he will run for mayor
```

각 메모리는 다음 정보를 가진다.

- 텍스트
    
- 생성 시간
    
- 최근 접근 시간
    
- 중요도
    

이 메모리는 계속 누적된다.

---

### 2) Memory Retrieval

행동을 결정할 때 모든 기억을 쓰지 않고 **중요한 기억만 검색한다.**

검색 점수:

```
Score =
recency +
importance +
relevance
```

즉

- 최근 기억
    
- 중요한 사건
    
- 현재 상황과 관련된 기억
    

이 세 가지를 합산한다.

---

### 3) Reflection

에이전트는 일정 시간이 지나면 **자기 경험을 분석한다.**

예:

```
Observation
Klaus read research paper
Klaus discussed research
Klaus studied sociology
```

→ Reflection

```
Klaus is highly dedicated to research
```

즉

**경험 → 추상적 자기 인식**

이 만들어진다.

---

### 4) Planning

에이전트는 하루 계획을 만든다.

예

```
7:00 wake up
8:00 breakfast
9:00 work
12:00 lunch
```

그리고 이 계획을 다시 세분화한다.

```
1pm research
1:10 read paper
1:30 write notes
```

즉

**장기 계획 → 세부 행동**
