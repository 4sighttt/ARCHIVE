# Context Rot (컨텍스트 부패)

**핵심 요약**

1. **Context Rot**은 AI가 긴 대화나 많은 정보를 처리할 때 **초기 정보가 점점 영향력을 잃는 현상**을 의미한다.
    
2. 원인은 모델의 **컨텍스트 창(context window)과 주의 분배(attention)** 구조 때문이다.
    
3. 토큰이 많아질수록 모델은 **모든 정보를 동일하게 유지하지 못하고 중요도를 재분배**한다.
    
4. 그 결과 **초반 지시, 조건, 사실**이 뒤쪽 정보에 의해 희석되거나 잊힌다.
    
5. 이 문제를 줄이기 위해 등장한 개념이 **컨텍스트 엔지니어링(context engineering)**이다.
    

---

### 1️⃣ Context Rot (컨텍스트 부패)

LLM은 모든 정보를 “기억”하는 방식이 아니라  
**현재 컨텍스트 안에서 확률적으로 계산**한다.

따라서 다음 문제가 발생한다.

- 입력 토큰이 많아질수록 **초기 토큰의 영향 감소**
    
- 중요 정보와 잡음 정보가 **동일한 수준으로 섞임**
    
- 긴 작업에서 **초기 조건을 잊어버림**
    

이 현상을 보통 **Context Rot** 또는 **Instruction Drift**라고 부른다.

연구와 엔지니어링 문서에서 반복적으로 언급되는 문제다.  
예: Anthropic, OpenAI 에이전트 설계 문서.

---

# 2️⃣ 해결 방법: Context Engineering

LLM 시스템을 설계할 때  
**어떤 정보를 언제 넣을지 설계하는 것**이다.

단순히 프롬프트를 잘 쓰는 것이 아니라  
**정보 흐름 자체를 설계하는 작업**이다.

---

# 3️⃣ 핵심 전략 ① Just-in-Time Context

모든 정보를 한 번에 넣지 않는다.

대신

```
필요한 순간 → 필요한 정보만 주입
```

예

잘못된 방식

```
규칙 50개 + 데이터 100개 + 작업 설명
```

좋은 방식

```
1단계: 작업 정의
2단계: 필요한 규칙만 제공
3단계: 필요한 데이터만 제공
```

이 방식은

- 토큰 절약
    
- 정확도 증가
    
- hallucination 감소
    

효과가 있다.

---

# 4️⃣ 핵심 전략 ② Harness Pattern

긴 작업을 **에이전트 시스템으로 분해**한다.

구조 예

```
Controller Agent
   │
   ├── Planner Agent
   ├── Research Agent
   └── Writer Agent
```

그리고 상태를 **JSON 형태로 기록**한다.

예

```json
{
  "goal": "blog post about AI cognition",
  "step": "research",
  "sources": [],
  "summary": ""
}
```

이 방식의 핵심

- **컨텍스트를 모델 내부가 아니라 외부 상태로 관리**
    
- 긴 작업에서도 **맥락 손실 방지**
    

---

# 5️⃣ 실전에서 가장 많이 쓰는 3가지

### 1. 압축 (Compression)

대화가 길어지면

```
전체 대화 → 핵심 요약
```

예

```
Conversation → summary → 새로운 컨텍스트
```

---

### 2. 구조화된 메모 (Structured Memory)

AI가 스스로 기록한다.

예

```
notes.md
facts.json
task_state.json
```

즉

**AI 외부 메모리 시스템**

---

### 3. 서브 에이전트 (Sub-agents)

복잡한 문제를 나눈다.

예

```
문제 분석 AI
자료 조사 AI
글 작성 AI
검증 AI
```

각각 작은 컨텍스트로 작업한다.

---

# 정리

AI가 정보를 “까먹는” 이유는

**모델 구조 때문이지 단순한 버그가 아니다.**

핵심 해결 방법은

1️⃣ 필요한 정보만 넣는 **JIT 컨텍스트**  
2️⃣ 상태를 외부에 저장하는 **Harness 구조**  
3️⃣ 작업을 나누는 **서브 에이전트 시스템**

이 세 가지다.

---

원한다면 다음도 설명할 수 있다.

- 왜 **100만 토큰 모델에서도 Context Rot이 발생하는지**
    
- **AI 에이전트 시대에서 Context Engineering이 핵심 기술이 되는 이유**
    
- 실제 **OpenAI / Anthropic 에이전트 설계 패턴**.
