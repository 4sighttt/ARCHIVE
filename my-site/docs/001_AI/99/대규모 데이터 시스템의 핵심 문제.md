==데이터 저장==  
==보다==  
==데이터 접근==

## 핵심 요약

1. 맞다. ==**대규모 데이터 시스템의 핵심 문제는 저장보다 검색==**이다.
    
2. 빅데이터 분야에서도 핵심 기술은 **Index(인덱싱)**이다.
    
3. 데이터가 커질수록 **전체 탐색은 불가능**해지기 때문에 인덱스가 필요하다.
    
4. 실제 시스템은 **여러 종류의 인덱스를 동시에 사용**한다.
    
5. AI 메모리 시스템도 결국 **데이터베이스 문제와 매우 유사한 구조**를 가진다.
    
6. 핵심 목표는 **O(n) 검색을 O(log n) 또는 O(1)에 가깝게 만드는 것**이다.
    

---

# 1️⃣ 빅데이터의 기본 문제

데이터가 작을 때

```text
데이터 100개
→ 전체 탐색 가능
```

데이터가 커지면

```text
데이터 1억개
→ 전체 탐색 불가능
```

그래서 필요한 것이

```text
Index
```

이다.

---

# 2️⃣ 인덱스의 개념

예

전화번호부

```text
이름 → 전화번호
```

정렬되어 있기 때문에

```text
전체 검색 X
```

가능하다.

시간 복잡도 예

|방식|시간|
|---|---|
|전체 검색|O(n)|
|이진 탐색|O(log n)|
|해시|O(1)|

---

# 3️⃣ 실제 빅데이터 시스템

대형 시스템은 보통 여러 구조를 결합한다.

예

```text
Raw Data
 ↓
Index Layer
 ↓
Query Engine
 ↓
Result
```

대표 기술

- Elasticsearch
    
- Apache Hadoop
    
- Apache Spark
    

---

# 4️⃣ AI 메모리 시스템과 동일한 문제

AI memory 시스템도 사실 구조가 같다.

```text
Memory store
 ↓
Index
 ↓
Retrieval
 ↓
LLM reasoning
```

차이점

|일반 DB|AI memory|
|---|---|
|정확 검색|의미 검색|
|SQL|semantic query|

---

# 5️⃣ 그래서 등장한 기술

AI용 인덱스

```text
Embedding
↓
Vector index
↓
Nearest neighbor search
```

대표 라이브러리

- FAISS
    
- ScaNN
    

이들은

```text
Approximate nearest neighbor (ANN)
```

알고리즘을 사용한다.

# 핵심 정리

빅데이터 시스템의 본질은

```text
데이터 저장
보다
데이터 접근
```

문제다.

그래서 거의 모든 대형 시스템의 핵심 기술은

```text
Index
```

이다.
