**핵심 상태 + 검색형 기억 + 외부 저장** 구조를 사용

**요약 (10줄 이하)**

1. AI 시스템은 보통 **3단계 기억 구조**로 설계된다.
    
2. **핵심 상태(Core state)**: 항상 유지되는 작은 정보.
    
3. 예: 목표, 규칙, 현재 작업 상태.
    
4. **검색형 기억(Retrieval memory)**: 필요할 때 검색해서 가져오는 지식.
    
5. 보통 **벡터 검색(RAG)** 같은 방식 사용.
    
6. **외부 저장(External storage)**: 데이터베이스·파일 같은 장기 저장소.
    
7. 대부분의 정보는 여기 있다가 **질문이 있을 때만 로드**된다.
    
8. 이렇게 해야 **컨텍스트 공간을 낭비하지 않는다.**
    
9. 실제 AI 시스템 대부분이 **이 구조를 변형해서 사용**한다.

**각 구조가 실제로 어떻게 작동하는지 (AI 내부 흐름)**을 설명

**요약 (10줄 이하)**

1. AI가 질문을 받으면 **먼저 작업 기억(컨텍스트)**에 현재 대화를 넣는다.
    
2. 질문을 분석해 **관련 정보가 필요한지 판단**한다.
    
3. 필요하면 **검색 메모리(RAG)**에서 관련 문서를 찾는다.
    
4. 검색된 정보 일부만 **컨텍스트에 추가로 로드**된다.
    
5. 모델은 **컨텍스트 안에 있는 정보만 보고 추론**한다.
    
6. 결과를 생성해 답변을 만든다.
    
7. 일부 시스템은 중요한 정보를 **요약하여 장기 저장**한다.
    
8. 대부분의 데이터는 **외부 저장소(DB, 파일)**에 남는다.
    
9. 다음 질문이 오면 이 과정이 **다시 반복**된다.

**요약 (10줄 이하)**

몰트북 글 작성자는 메모리를 **4가지 층으로 분류해 관리해야 한다**고 주장한다.

1. **Guaranteed memory**
    
    - 항상 로드되는 핵심 기억
        
    - 가치관, 목표, 현재 프로젝트 상태 등
        
    - 약 **5~15개만 유지**
        
2. **Probabilistic memory**
    
    - 필요할 때 **검색으로 불러오는 기억**
        
    - 과거 대화, 참고 자료 등
        
    - **수백~수천 개 가능**
        
3. **Hybrid memory**
    
    - **요약은 항상 로드**, 세부 내용은 검색
        
    - 프레임워크·지식 구조 같은 것
        
4. **External memory**
    
    - 데이터베이스·파일 시스템
        
    - 필요할 때만 **컨텍스트에 로드**
        

핵심 주장:  
**모든 정보를 같은 방식으로 기억하려는 것이 메모리 문제의 원인이다.**

**요약 (10줄 이하)**

1. 이 주장은 **직관적으로 상당히 설득력 있는 설계 아이디어**다.
    
2. 실제 컴퓨터 구조도 **캐시–RAM–디스크 같은 계층 구조**를 사용한다.
    
3. 인간 기억도 **작업 기억 / 장기 기억 / 외부 기록** 구조와 유사하다.
    
4. AI 시스템에서도 **Core state + Retrieval + DB** 구조가 일반적이다.
    
5. 따라서 **기본 방향 자체는 현실적인 설계와 일치한다.**
    
6. 다만 글은 **구체적 구현 방법이나 실험 검증은 제시하지 않는다.**
    
7. 특히 **Guaranteed memory를 무엇으로 정할지**가 실제로 가장 어려운 문제다.
    
8. 또한 **기억 중요도를 자동으로 판단하는 알고리즘**도 필요하다.
    
9. 그래서 이 글은 **완성된 기술이 아니라 설계 원칙에 가깝다.**
    

원하면 다음 단계에서  
**왜 “기억 중요도 판단”이 AI 메모리 설계에서 가장 어려운 문제인지** 설명할 수 있다.

**요약 (10줄 이하)**

1. AI가 **어떤 정보가 중요한지 객관적으로 판단하기 어렵다.**
    
2. 중요성은 **시간이 지나면서 바뀐다.**
    
3. 처음엔 사소한 정보가 **나중에 핵심 정보가 될 수 있다.**
    
4. 반대로 지금 중요해 보여도 **다시 사용되지 않을 수도 있다.**
    
5. AI는 보통 **빈도·최근성·유사도** 같은 간접 지표로 판단한다.
    
6. 하지만 이것들은 **진짜 의미적 중요성**을 완전히 반영하지 못한다.
    
7. 사용자의 **목표·맥락·미래 계획**을 알아야 정확히 판단 가능하다.
    
8. 그런데 이런 정보는 **명시적으로 주어지지 않는 경우가 많다.**
    
9. 그래서 메모리 시스템에서 **무엇을 저장할지 결정하는 문제**가 가장 어렵다.
    

원하면 다음 단계에서  
**현재 AI 시스템들이 실제로 사용하는 “기억 중요도 결정 방식 5가지”**를 설명할 수 있다.

**요약 (10줄 이하)**  
현재 AI 시스템에서 자주 쓰이는 **기억 중요도 판단 방식 5가지**:

1. **Frequency (빈도)**
    
    - 같은 정보가 여러 번 등장하면 중요하다고 판단.
        
2. **Recency (최근성)**
    
    - 최근에 사용된 정보일수록 중요하게 취급.
        
3. **Similarity relevance (유사도)**
    
    - 현재 질문과 의미적으로 가까운 정보 우선 검색.
        
4. **User signal (사용자 신호)**
    
    - 사용자가 명시적으로 저장하거나 강조한 정보.
        
5. **Task linkage (작업 연결성)**
    
    - 현재 진행 중인 작업과 직접 연결된 정보.
        

핵심 문제:  
이 방식들은 **완벽한 중요도 판단이 아니라 확률적 추정**이다.
